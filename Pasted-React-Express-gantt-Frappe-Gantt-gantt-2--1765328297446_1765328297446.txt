React + Express の生産管理アプリです。
/gantt 画面で Frappe Gantt を使って案件別ガントチャートを表示しています。

今の /gantt の挙動で、次の2つの問題が起きています：

1. 開始日・終了日・案件名で絞り込む入力フィールドが、
   画面表示直後の「最初の1秒ほどだけ」表示され、その後すぐ消えてしまう。
   最終的には、ページ上部には [←] [Today] [すべて] だけが残っている。

2. ガントチャート部分で、トラックパッドやマウスホイールで
   「縦方向のスクロール」をすると、
   画面が横方向に移動してしまう。
   （縦にスクロールしているのに、ガントが左右に動く挙動になっている）

ブラウザの HTML を見ると、

- ページ上部に
  「案件別ガントチャート / frappe-ganttで案件別タイムラインを表示 /
   [←] [Today] [すべて] / 開始日・終了日・案件名入力」
  というツールバーが一度は描画されるが、
- その後、別のコンポーネントによって上書きされているように見えます。
- また、ガントエリア内に [←] [Today] のボタンがもう1セット出ている部分もあります。

【やりたいこと】

### A. ツールバー（フィルター付き）が消えないようにする

1. /gantt ルートで実際に使われているページコンポーネントを特定してください。
   ルーティング定義（例：React Router の <Route path="/gantt" element={<... />} />）を確認し、
   どのコンポーネントが最終的な /gantt ページとして使われているか突き止めてください。

2. 開始日・終了日・案件名で絞り込む UI が定義されているコンポーネントを全検索で探してください。
   （キーワード：「開始日」「終了日」「案件名で絞り込み」など）

3. /gantt ページは、
   - 「案件別ガントチャート」というタイトル
   - 「frappe-ganttで案件別タイムラインを表示」というサブタイトル
   - その下に、次の要素を含む「1つのツールバー」
     - [←] ボタン
     - [Today] ボタン
     - [すべて] ボタン
     - 開始日 input type="date"
     - 終了日 input type="date"
     - 案件名フィルター input type="text"
   を描画する構造に統一してください。

4. それ以外の「古いツールバー」や「重複しているヘッダーコンポーネント」がある場合は、
   /gantt ページでは使わないように削除・統合してください。
   とくに、
   - ページ上部のフィルター付きツールバーが一度表示されたあと、
     データ読み込みやレイアウト再描画によって別のツールバーに差し替えられていないか
   を確認し、そのような置き換えが起きないようにしてください。
   条件付きレンダリング（loading 中だけ別ヘッダーを出す等）が原因なら整理してください。

5. 修正後、
   - ページをリロードしても
   - データ読み込み完了後も  
   ツールバー（開始日・終了日・案件名入力を含む）がずっと表示され続ける状態にしてください。

### B. スクロール挙動の修正（縦スクロールで横に動かないように）

1. ガントチャートを表示しているコンポーネントと、その親要素の CSS を確認し、
   .gantt-wrapper などガントエリアのコンテナが次のような基本スタイルになっていることを確認・修正してください：

   ----------------------------------------------------
   .gantt-wrapper {
     height: calc(100vh - 160px); /* ヘッダーの高さに応じて適宜調整 */
     overflow-y: auto;
     overflow-x: auto;
     position: relative;
   }
   ----------------------------------------------------

   body については次のようなイメージです：
   ----------------------------------------------------
   body {
     margin: 0;
     overflow: hidden;
   }
   ----------------------------------------------------

2. /gantt 関連のコンポーネントで、
   - onWheel ハンドラ
   - addEventListener("wheel", ...)
   - deltaY を横スクロールに変換している処理（例：element.scrollLeft += event.deltaY）
   があれば、すべて削除するか無効化してください。

   目的は：
   - 縦スクロール（ホイールや二本指上下）では、.gantt-wrapper の縦スクロールだけが動く
   - 横スクロール（Shift + ホイールや二本指左右）をしたときだけ、横方向に動く
   という標準のブラウザ挙動に戻すことです。

3. Frappe Gantt 側（ラッパーコンポーネントやカスタムフックを含む）でも、
   wheel イベントを横スクロールに変換しているようなコードがあれば無効化してください。

4. 修正後の期待挙動は次の通りです：
   - トラックパッドを「上下」に動かすと、縦方向にだけスクロールする
   - トラックパッドを「左右」に動かすか、Shift + ホイールを使ったときのみ横スクロールする
   - ページ全体（body）はスクロールせず、.gantt-wrapper 内だけがスクロールする

### C. フィルターとスクロールの両方が動作することの確認

最後に次を確認できる状態にしてください：

- /gantt ページを開いたとき、
  ヘッダー下に
  [←] [Today] [すべて] + 開始日 + 終了日 + 案件名フィルター
  が常に表示されていること（消えないこと）。
- 開始日・終了日・案件名を入力すると、Frappe Gantt に表示されるタスクが正しく絞り込まれること。
- トラックパッドやマウスホイールで上下スクロールしたときに、
  画面が横方向に勝手に移動せず、縦スクロールとして動作すること。

これらを満たすように、/gantt 関連のコンポーネント・レイアウト・CSS・スクロールイベント処理を整理して修正してください。
